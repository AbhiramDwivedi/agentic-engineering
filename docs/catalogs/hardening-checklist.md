# The Hardening Checklist

> **The decision it resolves:** what turns a working demo into a system that survives production?

> **Stub: planned for a later release.**

The cross-reference hub for the "just engineering" concerns that every pattern leans on but none
owns: retry and graceful degradation, the enforced observer rule, checkpointing and resumption,
observability and tracing, token and cost economics, and concurrency locks. Each gets a short
entry and a link to the chapter that argues it in full, so a pattern chapter can point here
instead of re-teaching the same rails. Draws from [Reliability](../reliability/index.md) and
[Operating in Production](../production/index.md).
